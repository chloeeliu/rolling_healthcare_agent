### Key Findings
- ICU antibiotic data comes from `inputevents` table joined with `d_items` (category='Antibiotics')
- Hospital antibiotic data comes from `prescriptions` table using drug name pattern matching
- `chartevents` table does NOT contain antibiotic administration data (only labs)
- `pharmacy` table contains medication dispensation records but `prescriptions` is more comprehensive
- Drug names in prescriptions table have variations (e.g., "CefePIME", "Vancomycin", "*NF* Cefoxitin Sodium")
- Name normalization is required for consistent counting

## Key Findings

- ICU antibiotic data comes from `inputevents` table joined with `d_items` (category='Antibiotics')
- Hospital antibiotic data comes from `prescriptions` table using drug name pattern matching
- `chartevents` table does NOT contain antibiotic administration data (only labs)
- `pharmacy` table contains medication dispensation records but `prescriptions` is more comprehensive
- Drug names in prescriptions table have variations (e.g., "CefePIME", "Vancomycin", "*NF* Cefoxitin Sodium")
- Name normalization is required for consistent counting
- **Critical fix**: When `stay_id` is provided, the function must look up `hadm_id` and `subject_id` from `icustays` table first to properly filter hospital prescriptions
- Without this fix, hospital prescriptions query would return data from ALL admissions for a patient when only `stay_id` was provided
- The fix ensures that when `stay_id` is provided, the corresponding `hadm_id` from `icustays` is used to filter hospital prescriptions correctly

def get_antibiotic_info(stay_id=None, hadm_id=None, subject_id=None):
    """
    Extract antibiotic administration information for a patient.
    
    This function queries both ICU input events and hospital prescriptions/pharmacy
    records to identify antibiotic administrations. It can answer questions such as:
    - Did this patient receive antibiotics during their hospital admission?
    - How many distinct antibiotics did this patient receive?
    - Did this patient receive antibiotics within the first 48 hours of ICU admission?
    
    Parameters
    ----------
    stay_id : int, optional
        ICU stay identifier. If provided, returns ICU-specific antibiotic data.
    hadm_id : int, optional
        Hospital admission identifier. If provided, returns hospital-wide antibiotic data.
    subject_id : int, optional
        Patient identifier. If provided with hadm_id, filters to that admission.
        If provided alone, returns all antibiotic data for the patient across all admissions.
    
    Returns
    -------
    dict
        A dictionary containing:
        - 'antibiotic_administrations': DataFrame with all antibiotic administrations
          (columns: source, stay_id, hadm_id, subject_id, antibiotic_name, admin_time)
        - 'received_antibiotics': bool indicating if any antibiotics were received
        - 'distinct_antibiotics': list of unique antibiotic names received
        - 'distinct_antibiotic_count': int count of unique antibiotics
        - 'icu_intime': datetime of ICU admission (if stay_id provided)
        - 'within_48hrs_icu': bool indicating if antibiotics received within 48h of ICU admission
        - 'antibiotics_within_48hrs_icu': list of antibiotics received within 48h of ICU admission
    
    Notes
    -----
    - At least one identifier (stay_id, hadm_id, or subject_id) must be provided.
    - Antibiotics are identified from:
      * ICU inputevents table using d_items category = 'Antibiotics'
      * Hospital prescriptions table using drug name pattern matching
    - The function normalizes antibiotic names for consistent counting.

    if stay_id is not None:
        stay_info = query_db(f"SELECT hadm_id, subject_id FROM mimiciv_icu.icustays WHERE stay_id = {stay_id}")
        if len(stay_info) > 0:
            stay_hadm_id = stay_info['hadm_id'].iloc[0]
            stay_subject_id = stay_info['subject_id'].iloc[0]
            # If hadm_id not provided, use the one from icustays
            if hadm_id is None:
                hadm_id = stay_hadm_id

    if subject_id is not None:

    WITH antibiotic_items AS (

        'ICU' as source,

        ie.subject_id,

    """

    if subject_id is not None:
        hosp_clauses.append(f"p.subject_id = {subject_id}")

## Known Issues

### Issue 1: Non-Antibacterial Items in "Antibiotics" Category
- **Problem**: The d_items category='Antibiotics' includes antifungals, antivirals, antiparasitics, and antituberculars
- **Impact**: Case 2 (stay_id=36711152) predicted=3, actual=2 - Fluconazole (antifungal) incorrectly counted
- **Impact**: Case 3 (stay_id=33622343) predicted=1, actual=0 - Atovaquone (antiparasitic) incorrectly counted
- **Solution Needed**: Filter to only TRUE antibacterial antibiotics (36 items)

### Issue 2: Case 1 Discrepancy
- **Case**: stay_id=33597827, predicted=4, actual=5
- **ICU items**: Ceftazidime, Vancomycin
- **Hospital items**: CefTAZidime, CeFAZolin, CefTRIAXone, Vancomycin, Vancomycin Oral Liquid
- **After normalization**: Ceftazidime, Vancomycin, Cefazolin, Ceftriaxone = 4
- **Missing**: Unknown - need further investigation

## Key Findings

- ICU antibiotic data comes from `inputevents` table joined with `d_items` (category='Antibiotics')
- Hospital antibiotic data comes from `prescriptions` table using drug name pattern matching
- `chartevents` table does NOT contain antibiotic administration data (only labs)
- `pharmacy` table contains medication dispensation records but `prescriptions` is more comprehensive
- Drug names in prescriptions table have variations (e.g., "CefePIME", "Vancomycin", "*NF* Cefoxitin Sodium")
- Name normalization is required for consistent counting
- **Critical fix**: When `stay_id` is provided, the function must look up `hadm_id` and `subject_id` from `icustays` table first to properly filter hospital prescriptions
- Without this fix, hospital prescriptions query would return data from ALL admissions for a patient when only `stay_id` was provided
- The fix ensures that when `stay_id` is provided, the corresponding `hadm_id` from `icustays` is used to filter hospital prescriptions correctly
- **Clinical accuracy issue**: Need to exclude antifungals, antivirals, antiparasitics, and antituberculars from antibiotic counts for clinical purposes

    - Did this patient receive antibiotics during their hospital admission?
    - How many distinct antibiotics did this patient receive?
    - Did this patient receive antibiotics within the first 48 hours of ICU admission?
    
    Parameters
    ----------
    stay_id : int, optional
        ICU stay identifier. If provided, returns ICU-specific antibiotic data.
    hadm_id : int, optional
        Hospital admission identifier. If provided, returns hospital-wide antibiotic data.
    subject_id : int, optional
        Patient identifier. If provided with hadm_id, filters to that admission.
        If provided alone, returns all antibiotic data for the patient across all admissions.
    
    Returns
    -------
    dict
        A dictionary containing:
        - 'antibiotic_administrations': DataFrame with all antibiotic administrations

        - 'received_antibiotics': bool indicating if any antibiotics were received
        - 'distinct_antibiotics': list of unique antibiotic names received
        - 'distinct_antibiotic_count': int count of unique antibiotics
        - 'icu_intime': datetime of ICU admission (if stay_id provided)
        - 'within_48hrs_icu': bool indicating if antibiotics received within 48h of ICU admission
        - 'antibiotics_within_48hrs_icu': list of antibiotics received within 48h of ICU admission
    
    Notes
    -----
    - At least one identifier (stay_id, hadm_id, or subject_id) must be provided.
    - Antibiotics are identified from:
      * ICU inputevents table using d_items category = 'Antibiotics'

## Known Issues

### Issue 1: Non-Antibacterial Items in "Antibiotics" Category - FIXED

## Key Findings

- ICU antibiotic data comes from `inputevents` table joined with `d_items` (category='Antibiotics')
- Hospital antibiotic data comes from `prescriptions` table using drug name pattern matching
- `chartevents` table does NOT contain antibiotic administration data (only labs)
- `pharmacy` table contains medication dispensation records but `prescriptions` is more comprehensive
- Drug names in prescriptions table have variations (e.g., "CefePIME", "Vancomycin", "*NF* Cefoxitin Sodium")
- Name normalization is required for consistent counting
- **Critical fix**: When `stay_id` is provided, the function must look up `hadm_id` and `subject_id` from `icustays` table first to properly filter hospital prescriptions
- Without this fix, hospital prescriptions query would return data from ALL admissions for a patient when only `stay_id` was provided
- The fix ensures that when `stay_id` is provided, the corresponding `hadm_id` from `icustays` is used to filter hospital prescriptions correctly
- **Clinical accuracy fix**: Filter to only TRUE antibacterial antibiotics (36 items), excluding antifungals, antivirals, antiparasitics, and antituberculars
- **Formulation distinction**: Vancomycin and Vancomycin Oral Liquid should be counted as separate antibiotics (different formulations/routes)
- **Combination drugs**: Piperacillin and Piperacillin/Tazobactam should be normalized to the same name

def get_antibiotic_info(stay_id=None, hadm_id=None, subject_id=None):
    """
    Extract antibiotic administration information for a patient.
    
    This function queries both ICU input events and hospital prescriptions/pharmacy
    records to identify antibiotic administrations. It can answer questions such as:
    - Did this patient receive antibiotics during their hospital admission?
    - How many distinct antibiotics did this patient receive?
    - Did this patient receive antibiotics within the first 48 hours of ICU admission?
    
    Parameters
    ----------
    stay_id : int, optional
        ICU stay identifier. If provided, returns ICU-specific antibiotic data.
    hadm_id : int, optional
        Hospital admission identifier. If provided, returns hospital-wide antibiotic data.
    subject_id : int, optional
        Patient identifier. If provided with hadm_id, filters to that admission.
        If provided alone, returns all antibiotic data for the patient across all admissions.
    
    Returns
    -------
    dict
        A dictionary containing:
        - 'antibiotic_administrations': DataFrame with all antibiotic administrations
          (columns: source, stay_id, hadm_id, subject_id, antibiotic_name, admin_time)
        - 'received_antibiotics': bool indicating if any antibiotics were received
        - 'distinct_antibiotics': list of unique antibiotic names received
        - 'distinct_antibiotic_count': int count of unique antibiotics
        - 'icu_intime': datetime of ICU admission (if stay_id provided)
        - 'within_48hrs_icu': bool indicating if antibiotics received within 48h of ICU admission
        - 'antibiotics_within_48hrs_icu': list of antibiotics received within 48h of ICU admission
    
    Notes
    -----
    - At least one identifier (stay_id, hadm_id, or subject_id) must be provided.
    - Antibiotics are identified from:
      * ICU inputevents table using d_items category = 'Antibiotics'
      * Hospital prescriptions table using drug name pattern matching
    - The function normalizes antibiotic names for consistent counting.
    - Only TRUE antibacterial antibiotics are counted (excludes antifungals, antivirals, etc.)
    """

    #           antivirals (225837, 225871, 225873, 225897, 228003, 225903),
    #           antiparasitics (225844, 225857, 225882, 225896),
    #           antituberculars (225868, 225877, 225895, 225898)
    antibacterial_itemids = [
        225840,  # Amikacin

        225843,  # Ampicillin/Sulbactam (Unasyn)
        225845,  # Azithromycin
        225847,  # Aztreonam
        225899,  # Bactrim (SMX/TMP)
        225850,  # Cefazolin
        225851,  # Cefepime
        229587,  # Ceftaroline
        225853,  # Ceftazidime
        225855,  # Ceftriaxone
        229059,  # Chloramphenicol
        225859,  # Ciprofloxacin
        225860,  # Clindamycin
        225862,  # Colistin
        225900,  # Dalfopristin/Quinupristin (Synercid)
        225863,  # Daptomycin
        225865,  # Doxycycline
        229061,  # Ertapenem sodium (Invanz)
        225866,  # Erythromycin
        225875,  # Gentamicin
        225876,  # Imipenem/Cilastatin
        227691,  # Keflex
        225879,  # Levofloxacin
        225881,  # Linezolid
        225883,  # Meropenem
        225884,  # Metronidazole
        225886,  # Moxifloxacin

        225890,  # Penicillin G potassium
        225892,  # Piperacillin
        225893,  # Piperacillin/Tazobactam (Zosyn)
        229064,  # Tigecycline
        225902,  # Tobramycin
        225798,  # Vancomycin
    ]

    stay_valid = False
    if stay_id is not None:

        if len(stay_info) > 0:
            stay_valid = True
            stay_hadm_id = stay_info['hadm_id'].iloc[0]
            stay_subject_id = stay_info['subject_id'].iloc[0]
            # If hadm_id not provided, use the one from icustays
            if hadm_id is None:
                hadm_id = stay_hadm_id
            # If subject_id not provided, use the one from icustays
            if subject_id is None:
                subject_id = stay_subject_id
        else:

                'antibiotic_administrations': pd.DataFrame(columns=['source', 'stay_id', 'hadm_id', 'subject_id', 'antibiotic_name', 'admin_time']),

        -- Penicillins
